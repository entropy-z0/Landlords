## 版本说明

### 版本号格式

MAJOR.MINOR.PATCH（主版本.次版本.补丁版本）

- **MAJOR**: 不兼容的 API 变更
- **MINOR**: 新增功能，向下兼容
- **PATCH**: 错误修复，向下兼容

### 发布流程

1. 更新版本号
2. 更新 CHANGELOG.md
3. 创建 Git 标签
4. 发布 GitHub Release
